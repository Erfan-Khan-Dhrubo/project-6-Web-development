Frequently Asked Questions

what is the difference between `var`, `let`, and `const`?
var is function-scoped, allows re-declaration, and is hoisted as undefined. let and const are block-scoped, don't allow re-declaration, and are hoisted but uninitialized. let supports re-assignment, while const doesn't,
what is the difference between `map()`, `forEach()`, and `filter()` ?
map(), forEach(), and filter() are used to work with arrays, but they differ in purpose. map() creates a new array by transforming each item, forEach() runs a function on each item but doesn’t return anything, and filter() creates a new array with items that meet a condition. For example, use map() to double numbers, forEach() for logging or other actions, and filter() to get specific items like even numbers.
explain arrow functions and how they are different from regular functions ?
Arrow functions in JavaScript are a compact way to write functions, introduced in ES6. They use => and are shorter than regular functions. Unlike regular functions, arrow functions don’t create their own this; they inherit this from the surrounding scope, making them great for callbacks and maintaining context. However, they cannot be used as constructors and don’t have access to the arguments object.
how JavaScript Promises work ?
avaScript Promises are a way to handle asynchronous operations, making it easier to manage tasks like API calls, timers, or file operations. A Promise represents a value that may be available now, in the future, or never. It has three states: pending (initial state), fulfilled (operation completed successfully), or rejected (operation failed).
how closures work in JavaScript?
In JavaScript, a closure is created when a function retains access to the variables in its outer scope, even after that outer function has finished executing. Closures allow inner functions to "remember" and use variables defined outside of their own scope.
